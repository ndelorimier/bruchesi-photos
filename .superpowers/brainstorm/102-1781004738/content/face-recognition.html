Où tourne la reconnaissance faciale ?

Décision clé : vie privée des enfants, coût, puissance requise sur le NAS

A

100 % local sur le NAS (CompreFace / DeepFace)

Container Docker open-source sur le Synology. Les photos ne quittent jamais le camp.

📸 Photo uploadée → NAS → CompreFace (Docker) → tag enfant → notif parent

Pour

  • Vie privée totale (LPRPDE/Québec 25)
  • 0 $ par photo
  • Fonctionne sans internet

Contre

  • NAS doit supporter Docker (DS923+, DS1522+...)
  • Setup plus technique
  • Précision dépend du modèle NAS
B

API Cloud (AWS Rekognition / Azure Face)

Les photos sont envoyées à un service cloud pour matching. Retour quasi-instantané.

📸 Photo uploadée → NAS → AWS Rekognition → match → tag → notif parent

Pour

  • Précision excellente
  • Pas de contrainte matérielle NAS
  • Rapide à intégrer

Contre

  • Photos d'enfants envoyées à AWS (~0,001 $/photo)
  • Dépend d'internet au camp
  • Enjeux légaux (Loi 25 Québec)
C

Synology Photos (face recognition intégré)

Utiliser la reconnaissance faciale native de Synology Photos comme moteur, et brancher l'app par-dessus via l'API Synology.

📸 Upload → Synology Photos → face detection natif → API → notif parent

Pour

  • Déjà intégré au NAS
  • 100 % local
  • Moins de setup Docker

Contre

  • API Synology moins flexible
  • Couplé à l'écosystème Synology
  • Moins de contrôle sur les seuils de précision
