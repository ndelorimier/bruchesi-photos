Design — Section 3 : Flux principaux

3 parcours critiques de l'app

① Enrôlement d'un campeur (avant + jour 1)

ADMIN 📥 Import CSV campeurs/parents → comptes créés → Email magic link envoyé à chaque parent PARENT 📧 Clique le lien → connecté → upload photo de son enfant ADMIN ✅ Valide la photo dans le panel → statut: partiel → CompreFace : sujet créé + 1ère photo de référence ajoutée EMPLOYÉ (Jour 1) 📸 Selfie Station : trouve l'enfant par nom → prend photo → CompreFace : 2ème photo de référence ajoutée → statut: confirmé

② Upload photo → Approbation → Tags

EMPLOYÉ 📤 Upload 1–50 photos (batch ou mobile) → statut: pending SYSTÈME → Queue async : CompreFace analyse chaque photo → photo_tags créés (campeur_id + confidence) APPROBATEUR 👁 Voit les photos pending avec tags proposés (noms + %) Peut corriger les tags, ajouter des oublis, retirer des erreurs ✅ Approuve (ou ❌ rejette) → statut: approved SYSTÈME → Photo visible dans l'album des semaines concernées

③ Notification et accès parent

SYSTÈME → Pour chaque photo approuvée : trouver les parents des campeurs tagués → Push notification Web Push (VAPID) : "📸 Nouvelle photo de Théo !" PARENT 📱 Reçoit la notif → ouvre l'app (magic link ou déjà connecté) Voit uniquement les photos de la semaine de son enfant Peut télécharger les photos individuellement ou en ZIP
Note sécurité : Un parent ne peut jamais accéder à des photos de semaines auxquelles son enfant n'est pas inscrit — le filtre est appliqué côté API, pas seulement côté frontend.
